
# switch to root user
```
sudo -i
```

# List all drives and their partitions
```
lsblk
```

# Now we need to copy the windows boot manager into the pop os EFI partition. To do so first create a directory namded windowsEFI inside the mnt directory
```
mkdir /mnt/windowsEFI
```

# Mount the windowEFI partition into the windowsEFI partition
```
mount /dev/nvme0n1p1 /mnt/windowsEFI
```

# once it is mounted... navigate into the windowsEFI/EFI directory
cd /mnt/windowsEFI/EFI

# then type ls to list the contents inside. you will see a folder named Microsoft which contains all the windows boot files
```
ls
```

# Now we need to copy the Microsoft folder into popOS EFI partition and cd to the EFI folder
```
cp -r Microsoft /boot/efi/EFI && cd /boot/efi/EFI
```

# type ls and you should see the Microsoft folder alongside pop os folder
```
ls
```

# cd to Microsoft boot
```
cd Microsoft/Boot
```

# type ls and you should find the essential windows files windows will need to boot. make sure that the **bootmgfw.efi** file is present inside this directory as it is required to load windows.

# Now type this command to get the full location of the windows boot files.
```
pwd
```

# you should be seeing this... make sure you copy the path
```
/boot/efi/EFI/Microsoft/Boot
```

# Next, we need to create a windows config file

```
nano /boot/efi/loader/entries/windows.conf
```

# then add the following lines
```
title Windows Boot Manager
efi /boot/efi/EFI/Microsoft/Boot/bootmgfw.efi
```

# Now we need to modify the pop os loade configuration file
```
nano /boot/efi/loader/loader.conf
```

# And add this line
```
timeout 30
```

# now you can restart
```
reboot now
```
